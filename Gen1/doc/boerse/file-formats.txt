1) Event = Handelstag
Idee XML pro Handelstag mit den zu testenden Aktien

<event>
	<kurs>
		<timestamp>2013-11-29 17:45 CET</timestamp>
		<name>allianz</name>
		<min>95.4</min>
		<max>97.49</max>
		<value>96.2</value>
		<currency>EURO</currency>
	</kurs>
	<kurs>
		...
	</kurs>
</event>

Es sind XML Kommentare zugelassen: 
	<!-- ein Kommentar -->
Einschränkung/Vereinfachung: sie müssen alleine in einer Zeile stehen.
Folgendes geht auch
	<min></min>
bzw.
	<min/>

3) Programm unter Test
3.1) Aufruf
	<program> -data <data-directory> -event <event-id> [-window <size>]
Beispiel:
	trader_123456.exe -data c:/temp/boerse/ -event 1003700

D.h. in der (read-only) Directory <data-directory> befinden sich die Dateien:
	event-1003450.xml
	...
	event-1003700.xml
	portfolio-1003700.xml
	
Diese enthalten 
a) das Portfolio zum (abstrakten) Zeitpunkt 1003700.
b) Mindestens 250 Kursangaben bis zum Zeitpunkt 1003700.

3.2) Ausgabe auf standard out 
Ist zeilenorientiert und enthält den Handelsvorschlag.
	<aktienname>: <+-stueck>
Beispiel
	allianz: +20
Bedeutet, dass das Programm 20 Allianz Aktien kaufen will.

4) Framework
4.1) Aufruf
	onetest -testee <program-under-test> -data <data-directory> [-window <size>] -tests <days> [-name <aktien-input-file>]
Beispiel:
	onetest -testee "C:\Users\wrossner\boerse\trader_0.exe" -data C:\Users\wrossner\boerse\tmp\ -window 1 -tests 20 -name "C:\Users\wrossner\git\autostock\Gen1\doc\boerse\dax\allianz.txt"
